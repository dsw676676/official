## 2. 📊 文件使用建议

### 2.1. 按使用频率排序

| 使用频率 | 文件类型 | 文件示例 | 建议 |
|----------|----------|----------|------|
| **高频** | 核心数据 | 考试类型对比.md | 每日查看，快速查询 |
| **中频** | 官方资源 | 官方网站索引.md | 定期检查，更新收藏 |
| **中频** | 时政新闻 | 每日简报.md | 每日阅读，积累素材 |
| **低频** | 详细资料 | 各省详细文件 | 按需查阅，深度研究 |

### 2.2. 按token消耗排序

| 消耗等级    | 文件类型 | 平均大小   | 读取建议      |
| ------- | ---- | ------ | --------- |
| **低消耗** | 核心数据 | 2-4KB  | 可频繁读取     |
| **低消耗** | 官方资源 | 2KB    | 可定期读取     |
| **中消耗** | 时政新闻 | 5-10KB | 每日读取一次    |
| **高消耗** | 详细资料 | 10KB+  | 按需读取，避免频繁 |

## 3. 🔧 系统维护指南

### 3.1. 定期更新任务

| 更新周期    | 更新内容 | 操作说明      |
| ------- | ---- | --------- |
| **每日**  | 时政新闻 | 自动生成，无需手动 |
| **每月**  | 官方链接 | 检查链接有效性   |
| **每季度** | 备考资源 | 更新推荐资料    |
| **每年**  | 核心数据 | 更新招录政策数据  |

### 3.2. 数据更新流程

1. **收集信息**：从官方网站获取最新政策
2. **更新核心**：修改核心数据文件
3. **更新资源**：更新官方链接和备考资源
4. **存档历史**：将旧数据移动到详细资料/历史存档

## 4. 💡 高效使用技巧

### 4.1. token节省策略

1. **优先读取核心数据**：大小仅2-4KB，效率最高
2. **避免频繁读取详细资料**：单个文件10KB+，消耗大
3. **使用摘要和索引**：通过摘要定位，再按需查看详细
4. **批量操作**：一次性读取相关文件，减少单独读取

### 4.2. 查询优化建议

1. **先摘要后详细**：先用摘要筛选，再查看详细
2. **建立个人索引**：在Obsidian中建立个人笔记链接
3. **使用搜索功能**：利用Obsidian搜索快速定位
4. **定期整理**：删除不再需要的临时文件

## 5. 🛠️ 故障排除

### 5.1. 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| **文件找不到** | 路径错误 | 检查文件夹结构 |
| **链接失效** | 网站更新 | 更新官方网站索引 |
| **数据过时** | 政策变化 | 更新核心数据文件 |
| **token消耗大** | 读取详细文件 | 改用核心数据文件 |

### 5.2. 系统恢复

如果系统出现问题，可以：
1. **检查文件夹结构**：确保符合标准结构
2. **验证核心文件**：确保核心数据文件存在
3. **重新生成索引**：根据需要重新创建索引文件
4. **联系技术支持**：如有需要可寻求帮助
